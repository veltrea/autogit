# AutoGit

AutoGitは、**「手元の開発プロジェクトを、非公開リポジトリ（Private Repository）へ自動的に同期し、履歴を安全に残すこと」**に特化したバックグラウンド・ツールです。

「Gitのコマンドは難しいけれど、作業の跡を日記のように残したい」「万が一のためにGitHubへ自動バックアップしたい」という方のための「おまかせ防衛ツール」です。

## 特徴

- 暗黙的なバージョン管理：ファイルを保存するだけでGitHubにバックアップ。
- チャタリング防止：連続した書き込みが落ち着くまで待機してコミット。
- メモ帳ユーザー向け：複雑なGitコマンドを覚える必要なし。

## 導入方法

1. **ビルド**:
   ```bash
   cargo build --release
   ```

2. **グローバルインストール (推奨)**:
   ビルドされたバイナリをパスの通った場所（`/usr/local/bin`）に配置することで、どのディレクトリからでも `autogit` コマンドが使えるようになります。
   ```bash
   # 実行権限を付与してシステムパスへ移動 (Mac/Linux)
   chmod +x target/release/autogit
   sudo cp target/release/autogit /usr/local/bin/
   ```

## 🚀 開発の背景 (Motivation)
このツールは、高度なAIコーディング・エージェント（Google AntiGravityなど）とのペアプログラミング体験をより円滑にするために生まれました。

1. **同期の安定化**: AIエージェント単体では稀にGitHubとの同期が不安定になり、不毛な再試行ループや思考の停滞（ループ的思考）が発生することがあります。AutoGitに同期を任せることで、AIは「コードを書くこと」に専念でき、作業が止まるのを防ぎます。
2. **トークン消費の抑制**: AIエージェントがGitコマンドの実行状況やエラー解決に思考を割く必要がなくなるため、余計なコンテキストを消費せず、結果としてトークン効率の向上が期待できます。
3. **安全な自動化**: AIの手が滑って機密情報をプッシュしてしまう前に、ローカルの専用ガードが最終的なブレーキをかけます。

### 🛡️ プライベート・ファーストの思想 (Private-first Philosophy)
AutoGitは、あなたが監視するディレクトリが**「非公開（Private）リポジトリ」と同期されていること**を前提としています。

- **あえて「緩い」管理**: AutoGitのセーフガード（機密情報検知）があるため、開発中は `.gitignore` をガチガチに固める必要はありません。エラーログや一時ファイルを「とりあえず」残しておいても、AutoGitが公開を阻止してくれます。
- **公開用とは分ける**: 公開リポジトリを直接監視対象にするのではなく、まずは「非公開の実験場」としてAutoGitを活用してください。

> [!WARNING]
> **重要な安全上の警告: .gitignore の限界**
> Gitの仕組み上、`.gitignore` はワークスペースに一つしか存在できません。そのため、「自分用の非公開リポジトリには含めたいが、GitHubの公開リポジトリには絶対に入れたくないファイル（例: 個人的な設計メモや秘密の TODO）」を `.gitignore` だけで管理するのは不可能です。
>
> この問題を解決するため、AutoGitは公開時専用の除外ルール **`.autogit-publish-ignore`** を導入しています。

## ✨ 主な機能 (Real-time Sync & Safety)

## 使い方

### 1. 監視したいディレクトリへ移動
同期したいプロジェクトのフォルダへ移動します。
```bash
cd /path/to/your/project
```

### 2. 保存先のリポジトリを登録
どのフォルダからでも `autogit` と打つだけで呼び出せます。
```bash
autogit link-repo https://github.com/your-username/your-repo.git
```

### 3. 監視と自動同期の開始
```bash
autogit watch . --interactive
```

> [!IMPORTANT]
> **このターミナルを閉じないでください！**
> AutoGitはあなたがコードを書いている間、このウィンドウでずっと見守っています。ターミナルを閉じると自動同期も止まってしまいます。作業が終わるまで、ウィンドウを最小化して隅に置いておいてください。

### 🌍 公開リポジトリへの安全な書き出し
開発が進み、プロジェクトを世界に公開したい準備ができたら、以下のコマンドを実行します。

```bash
autogit publish-init https://github.com/user/public-repo.git
```

これにより、`publish_public.sh` が生成されます。このスクリプトを実行すると：
1. 現在のファイルを一時フォルダにコピー（`.git` 履歴は除外されます）。
2. **`.autogit-publish-ignore` に記載されたファイルを削除。**
3. 新しく「クリーンな」リポジトリとして初期化。
4. 指定した公開リポジトリへ強制プッシュ。

> [!TIP]
> `publish-init` を実行すると、標準的な除外設定が含まれた `.autogit-publish-ignore` が自動生成されます。公開前に必ずこのファイルを開き、秘密にしたいファイルやフォルダが網羅されているか確認してください。

### 💬 表示される「質問」の全パターン
`--interactive` モードや監視開始時に、AutoGitは以下の3つのタイミングであなたに質問（確認）をします。これらはすべて**正常な動作**です。

#### 1. 監視開始時の「安全確認」
リポジトリの状態に応じて、開始直後に以下のいずれか、または両方の質問が出ることがあります。
- **公開状態チェック**: `[?] このまま監視を開始しますか？ (y/n)`
  （リポジトリがGitHubで「公開（Public）」状態の時に表示されます。冒頭に `[!] 警告` と出ます）
- **命名規則チェック**: `[?] このまま監視を開始しますか？ (y/n)`
  （リポジトリ名に `-private` が含まれていない時に `[!] 注意` として表示されます）

#### 2. 同期前の「ファイル選択」
ファイルが保存されると、数秒後に選択画面が出ます。
```text
--- どの変更をGitHubへ公開（ステージ）しますか？ ---
[?] スペースキーで選択・解除、Enterで確定してください
 > [x] src/main.rs
   [x] README.md
```
- **操作**: `↑/↓` で移動、`スペース` で選んで `Enter` で確定。

#### 3. 機密情報検知時の「安全警告」
もしファイル内にAPIキーや本名などが含まれていると、同期直前に警告が出ます。冒頭に `[!] 警告` と表示されます。
- **警告**: `[?] これらのファイルをそのままGitHubへ公開してもよろしいですか？ (y/N)`
